# 煤炭管控平台 — Windows Server 2016 部署指南

本文档包含两种用法：**Docker Compose 一键栈**（与仓库根目录 `docker-compose.yml` 一致）与 **Windows 原生安装**（Java + Nginx + JAR）。

**现场部署策略**：**优先**整栈放在 **Docker** 里部署与联调；**仅当**现场无法完成 Docker 的安装或运行（授权、内核版本、安全策略、运维规范等）时，再改用下文 **「一、软件安装清单」** 起的 **Windows 原生** 路径。两套方式业务功能一致，差异在打包与运维命令。

---

## Docker Compose 部署（推荐）

### 前置条件

- 已安装 **Docker Desktop**（Windows）或 **Docker Engine + Compose Plugin**（Linux）

### 启动

在仓库根目录（含 `docker-compose.yml` 的目录）执行：

```bash
docker compose up -d --build
```

### 访问地址

| 入口 | 默认 | 说明 |
|------|------|------|
| **Web（前端 + 反代）** | `http://localhost`（端口由环境变量 `WEB_PORT` 控制，默认 **80**） | `web` 容器内 Nginx 托管 Vue 静态页，并把 **`/api/`** 转发到后端 **`http://server:8080/api/`**，**`/ws/`** 转发到 **`http://server:8080/api/ws/`**（IoT WebSocket） |
| **后端（直连调试）** | `http://localhost:8080`（`SERVER_PORT`，默认 **8080**） | 一般经 Web 反代访问即可；直连时路径仍为 **`/api/...`** |
| **PostgreSQL** | 主机 `localhost:5432`（`POSTGRES_PORT`） | 便于本机用工具连库；生产若不需要对外暴露，可去掉 `ports` 映射 |

### KEPServer 在宿主机（与容器内后端通信）

`docker-compose.yml` 里后端默认：

`PLATFORM_IOT_KEPSERVER_ENDPOINT=opc.tcp://host.docker.internal:49320`

在 **Windows Docker Desktop** 下，`host.docker.internal` 通常指向宿主机，容器内的 Spring 可访问本机已启动的 KEPServer。

**Linux 服务器** 若没有 `host.docker.internal`，请任选其一：

- 把 `PLATFORM_IOT_KEPSERVER_ENDPOINT` 改成 **宿主机真实内网 IP**，例如 `opc.tcp://192.168.1.50:49320`；或  
- 在 `server` 服务下增加：`extra_hosts: ["host.docker.internal:host-gateway"]`（Docker 20.10+），再继续使用 `host.docker.internal`。

接入现场 OPC 时，同时设置 **`PLATFORM_IOT_KEPSERVER_ENABLED=true`**，并在库表 **`iot_tag`** 中配置与停送电一致的 **`source_path`**（参见 `deploy/现场对接数据清单.md`）。

### 预置 IoT 点位 SQL（可选）

在 Postgres 容器内执行种子脚本（用户名、库名需与 compose 中 `POSTGRES_USER`、`POSTGRES_DB` 一致，默认 `coal_user` / `coal_platform`）：

```bash
docker cp deploy/sql/seed-iot-tags.sql coal-platform-postgres:/tmp/seed.sql
docker exec -it coal-platform-postgres psql -U coal_user -d coal_platform -f /tmp/seed.sql
```

### 常用命令

```bash
docker compose ps
docker compose logs -f server
docker compose logs -f web
docker compose down
```

### 若现场无法安装或使用 Docker

改走 **Windows 原生安装**：从下文 **「一、软件安装清单」** 起按步骤安装 Java、PostgreSQL（及 TimescaleDB）、Nginx，再部署 JAR 与前端静态资源。网络与 OPC 配置思路与 Docker 节相同，仅「后端进程在本机、数据库在本机或指定主机」而非容器内访问。

## 一、软件安装清单

### 1. Java 17（运行 Spring Boot 后端）

- 下载地址：https://adoptium.net/temurin/releases/?version=17&os=windows&arch=x64
- 选择 `.msi` 安装包，安装时勾选 "Set JAVA_HOME" 和 "Add to PATH"
- 验证：打开 CMD 执行 `java -version`，应显示 openjdk 17.x

### 2. PostgreSQL 15（数据库）

- 下载地址：https://www.enterprisedb.com/downloads/postgres-postgresql-downloads
- 选择 Windows x86-64 版本，一路下一步
- 记住设置的 postgres 用户密码（默认端口 5432）
- 安装完成后用 pgAdmin 创建数据库 `coal_platform`

### 3. TimescaleDB 扩展（时序数据能力）

- 下载地址：https://docs.timescale.com/self-hosted/latest/install/installation-windows/
- 运行安装程序，它会自动检测已安装的 PostgreSQL 并添加扩展
- 安装完成后，在 pgAdmin 中对 `coal_platform` 数据库执行：
  ```sql
  CREATE EXTENSION IF NOT EXISTS timescaledb;
  ```
- 然后执行 `server/src/main/resources/db/init-timescaledb.sql` 中的建表脚本

### 4. Nginx for Windows（前端托管 + 反向代理）

- 下载地址：https://nginx.org/en/download.html（选 Windows 稳定版）
- 解压到 `D:\platform\nginx\`
- 将 `deploy/nginx.conf` 复制到 `D:\platform\nginx\conf\nginx.conf`

### 5. KEPServer EX（已有）

- 确认 OPC-UA 端点已启用，默认地址 `opc.tcp://127.0.0.1:49320`
- 在 KEPServer 配置中确认通道（Channel）和设备（Device）已配置好

**与现场「智能停送电」系统对齐**：若同现场已部署 `PowerOutageManagement\智能停送电`，其 `OPCUA_ENDPOINT`、账号、安全策略及 `OPCUA_NODEIDS` 写法已调通，管控平台应复用同一套 KEPServer 连接参数；点位标识符写入本系统 `iot_tag.source_path`（详见 `deploy/现场对接数据清单.md` 章节「〇、与智能停送电项目对接方式对齐」）。

## 二、构建打包

### 前端构建
```bash
cd web
npm install
npm run build
```
构建产物在 `web/dist/` 目录

### 后端构建
```bash
cd server
mvnw.cmd clean package -DskipTests
```
产物在 `server/target/platform-server-0.0.1-SNAPSHOT.jar`

## 三、部署目录结构

```
D:\platform\
├── server.jar                     # 后端 JAR（从 target 复制过来并重命名）
├── application-prod.yml           # 生产环境配置（修改数据库密码等）
├── coal-platform-service.xml      # WinSW 服务配置
├── coal-platform-service.exe      # WinSW 可执行文件（从 GitHub 下载并重命名）
├── nginx\                         # Nginx
│   ├── nginx.exe
│   └── conf\nginx.conf
└── web\dist\                      # 前端构建产物
```

## 四、配置步骤

### 1. 初始化数据库
```bash
# 使用 pgAdmin 或 psql 连接 PostgreSQL
psql -U postgres -d coal_platform -f init-timescaledb.sql
```

### 2. 修改生产配置
编辑 `application-prod.yml`，修改：
- `spring.datasource.password`：PostgreSQL 的 postgres 用户密码
- `platform.iot.kepserver.endpoint`：KEPServer 的 OPC-UA 地址
- `platform.iot.kepserver.channel-prefix`：KEPServer 中配置的通道前缀

### 3. 注册 Windows 服务
```bash
# 管理员身份打开 CMD
cd D:\platform
coal-platform-service.exe install
coal-platform-service.exe start
```

### 4. 启动 Nginx
```bash
cd D:\platform\nginx
start nginx
```

### 5. 验证
- 打开浏览器访问 `http://localhost`，应看到管控平台首页
- 访问 `http://localhost/api/iot/provider/status`，应看到 KEPServer 连接状态

## 六、常见问题

### Q: 启动后端报 "连接数据库失败"
A: 检查 PostgreSQL 服务是否启动，检查 application-prod.yml 中的密码是否正确

### Q: 前端页面空白
A: 检查 Nginx 配置中 `root` 路径是否指向正确的 `dist` 目录

### Q: OPC-UA 连接失败
A: 检查 KEPServer 是否运行，端口 49320 是否开放，防火墙是否放行

### Q: 实时数据不更新
A: 检查数据库 `iot_tag` 表中是否有已启用的点位，`source_path` 字段是否与 KEPServer 中的节点路径一致

### Q: Docker 里后端连不上本机 KEPServer
A: Windows 确认 `PLATFORM_IOT_KEPSERVER_ENDPOINT` 使用 `opc.tcp://host.docker.internal:49320` 且本机防火墙放行 49320；Linux 改用宿主机 IP 或配置 `extra_hosts`。确认已设 `PLATFORM_IOT_KEPSERVER_ENABLED=true` 并重启 `server` 容器：`docker compose up -d server`。
